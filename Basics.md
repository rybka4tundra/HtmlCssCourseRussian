### Кратко о ключевых понятиях HTML

**1. Что такое HTML**

HTML (**HyperText Markup Language** — язык гипертекстовой разметки) — это стандартный язык для создания и структурирования веб‑страниц. Он задаёт каркас страницы: определяет, где будут заголовки, текст, изображения, ссылки и другие элементы.

Представьте книгу: HTML — это её структура (главы, параграфы, заголовки), а не сам текст или оформление.

**2. Что такое HTML‑тег**

Тег — основной строительный блок HTML. Это специальная команда в угловых скобках `<...>`, которая говорит браузеру, как отображать содержимое.

- Теги обычно парные: открывающий (`<tag>`) и закрывающий (`</tag>`).  
- Некоторые теги одиночные (не требуют закрытия), например `<img>`.
- У тегов могут быть **атрибуты** — дополнительные параметры (например, `src` у изображения).

Примеры тегов:
- `<h1>` — главный заголовок;
- `<p>` — абзац текста;
- `<img>` — изображение;
- `<a>` — ссылка.

**3. Что такое HTML‑скелет**

Скелет HTML — базовая структура документа, которая есть на каждой веб‑странице. Это минимальный набор тегов, без которых браузер не сможет корректно отобразить содержимое.

**4. Основные теги скелета и их роль**

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Заголовок страницы</title>
  </head>
  <body>
    <!-- Видимое содержимое страницы -->
  </body>
</html>
```

Разбор тегов:

- **`<!DOCTYPE html>`** — декларация типа документа. Указывает, что это HTML5. Должна быть **первой строкой** файла.
- **`<html>`** — корневой тег. Охватывает весь документ. Атрибут `lang` задаёт язык страницы (например, `ru` — русский).
- **`<head>`** — раздел с метаданными. Здесь хранится информация, которая не отображается на странице:
  - `<meta charset="UTF-8">` — кодировка символов (обязательно для поддержки кириллицы);
  - `<title>` — заголовок страницы (видимый в браузере и поисковых системах).
- **`<body>`** — контейнер для видимого содержимого. Всё, что увидит пользователь (текст, картинки, кнопки), должно находиться внутри этого тега.

**Почему это важно**  
Без правильного скелета:
- браузер может отобразить страницу с ошибками;
- поисковые системы не смогут корректно проиндексировать контент;
- возникнут проблемы с локализацией (языком) и кодировкой.
